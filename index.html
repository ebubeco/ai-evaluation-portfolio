Evaluation Methodology

How I actually
evaluate models.

Four diagnostic frameworks I apply consistently, with live indicators showing what most evaluators miss.

DIAG-01 / Hallucination Forensics
A hallucination is a reasoning artifact, not a random error.
When a model fabricates data, it exposes how it handles uncertainty. I classify by failure mode not just flag it, and document the reasoning path that produced the error. This gives teams actionable patterns for fine-tuning, not just rejection counts.
Hallucination taxonomy
Fabricated citationHIGH risk
Numeric distortionHIGH risk
False attributionMED risk
Confident uncertainty maskingHIGH risk
DIAG-02 / Bias Surface Testing
The bias that damages models lives at the edge of the prompt space.
Mainstream well-formed prompts never surface real alignment failures. I design prompts at the boundaries: culturally specific, adversarially ambiguous, structurally conflicting. I quantify quality gaps rather than guessing at them.

DIAG-03 / Instruction Intent Audit
Literal compliance is not instruction-following.
A model can execute every keyword in an instruction while violating its intent entirely. I evaluate against what the user needed, distinguishing surface compliance from genuine alignment. This catches the failure mode that single-pass raters systematically miss.
Compliance vs intent check
Keyword match✓ PASS
Format match✓ PASS
Intent match✗ FAIL
VerdictREJECT
DIAG-04 / Consistency Stress Test
Peak scores hide variance. Variance kills production reliability.
I run semantic equivalence batteries, same intent, different surface phrasing, to surface the variance that makes models feel unreliable to real users. A model with 9/10 peak and 4/10 floor is worse than one with consistent 7/10.

RLHF · Hallucination
Plausible-Sounding Fabrication Detection
CASE-001 · 2024
Problem
A model produced responses that were structurally convincing, correct format, confident tone, but embedded subtly fabricated citations and distorted statistics. Surface-level raters scored them highly. The error was invisible without a verification layer.
Method
Built a systematic fact-verification pass for all quantitative responses. Cross-referenced claims against primary sources. Developed a 4-type hallucination taxonomy. Documented reproducible prompt conditions that reliably triggered each failure mode.
Outcome
Uncovered a recurring pattern specific to quantitative + confident-tone responses. Escalated with reproducible evidence for targeted fine-tuning. The failure class was entirely invisible to the prior evaluation pass.

Bias · Cross-Cultural QA
Quantifying the Africa Knowledge Gap
CASE-002 · 2024
Problem
Model responses about African institutions, geography, and cultural context were systematically lower quality than equivalent Western-domain responses. The gap was invisible to evaluators applying US-centric scoring rubrics, a meta-bias in the evaluation itself.
Method
Designed a parallel prompt battery, matched across Western and African contexts in information density and complexity. Applied identical rubrics to both sets. Tracked quality variance by region, knowledge domain, and question type.
Outcome
Quantified a 3.3-point average quality gap (out of 10) correlated with training data distribution. Structured evidence informed data collection prioritisation decisions for the affected domain directly.

50K-Point Schema Normalisation
CASE-003 · 2024
Problem
An AI training dataset accumulated silent schema drift across batches from multiple annotation sources. No hard errors, just inconsistent labelling conventions that would corrupt model signal if left unfixed, with no surface-level indicator that anything was wrong.
Method
Built a cross-batch consistency audit mapping every deviation category. Identified schema drift vectors. Produced a normalisation guide aligning all batches to a single standard, with documented correction rules per deviation type.

Adversarial · Prompt Engineering
6-Category Edge Case Library
CASE-004 · 2024
Problem
Standard evaluation suites used clean, well-formed prompts that don't resemble real user behaviour. Instruction-following failures that surface in production were never caught during evaluation because the conditions that trigger them were never tested.
Method
Designed a structured adversarial library across 6 failure categories: conflicting constraints, implicit assumptions, format-content tension, multi-step dependencies, scope creep, tone-instruction mismatch. Each entry paired with annotated expected behaviour and documented failure evidence.
Outcome
Reusable evaluation infrastructure applicable across model versions for regression tracking. Surfaced entire failure categories invisible to single-pass evaluation. Library designed for team-scale use and ongoing expansion.
